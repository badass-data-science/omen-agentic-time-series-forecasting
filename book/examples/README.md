# Book Example Datasets

Every named dataset used in "Agentic Time Series Forecasting for Supervillains" — Secret
Lab™ Mojito Inventory, Death-Ray Revenue, Henchman Costume Dry-Cleaning
Bills, Secret Lab™ Power Consumption, and the rest — is generated by
`generate_book_datasets.py`, deterministically, from a fixed seed. Every
tool result quoted in the book comes from running the real Omen tools
against the output of this script, not from hand-typed or estimated
numbers.

## Regenerate everything

```bash
cd book/examples
python generate_book_datasets.py
```

Writes every dataset as a CSV into `./data/`. Requires Omen installed
(`pip install -e ".[all]"` from the project root — `omen.data_prep`
itself only needs `numpy`/`pandas`, but you'll want the rest to actually
run the tools each chapter calls).

Regenerate just one dataset:

```bash
python generate_book_datasets.py --only deathray_revenue
```

See the docstring on each function in `generate_book_datasets.py` for
which chapter(s) use that dataset and why it's shaped the way it is.

## Why this file exists

Chapter drafts for this book are held to the same standard the rest of
the Omen project holds its own claims to: a number doesn't go in the
book until it's been produced by a real run against real data. That
promise is only honest if a reader can rerun the same generation and get
the same series back — which is the entire job of this script.
